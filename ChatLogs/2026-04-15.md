# 2026-04-15 Combo Follow-up

## 작업 목적
- `Slash1 -> Slash2`가 왜 불안정한지 코드 문제인지, Animator 설정 문제인지 분리해서 확인
- 원인이 확인되면 바로 콤보 체인 안정화 작업 진행

## 확인 결과
- 결론은 설정만의 문제가 아니라 코드와 Animator 전이 타이밍이 같이 어긋난 상태였다.

### 코드 쪽 원인
- `PlayerInputHandler`의 `AttackPressed`는 프레임성 입력이라, 콤보 허용 구간보다 조금 빨리 누르면 입력이 사라질 수 있었다.
- `PlayerLightAttackState`는 콤보 입력을 받아도 다음 공격 상태 전환을 공격 종료 시점까지 미뤘다.
- 그래서 Animator가 1타 상태 안에서 2타로 이어져야 하는데, 코드 타이밍은 너무 늦게 반응했다.

### Animator 쪽 원인
- `Assets/PlayerAnimator.controller`를 직접 확인한 결과:
  - `Slash1 -> Slash2` 전이 조건은 `Attack` 트리거
  - 그런데 `Exit Time`이 `1.2165`로 되어 있어 실제 1타 콤보 타이밍과 맞지 않았다.
  - `Slash1 -> Idle`도 `0.6631635`로 빨라서 2타 입력 전에 빠질 여지가 있었다.

## 이번에 적용한 수정

### 코드
- `PlayerInputHandler`
  - 공격 입력 버퍼 추가
  - `HasBufferedAttack`, `ConsumeBufferedAttack` 추가
- `PlayerStateMachine`
  - `combatTuning.inputBufferWindow`를 읽는 `InputBufferWindow` 추가
  - 공격 버퍼 접근 헬퍼 추가
- `PlayerIdleState`, `PlayerMoveState`, `PlayerDodgeState`
  - 즉시 프레임 입력 대신 버퍼 입력 소비 방식으로 변경
- `PlayerLightAttackState`
  - 콤보 입력을 버퍼에서 읽도록 변경
  - `comboWindowEndTime`에 도달하면 바로 다음 공격 상태로 전환하도록 변경

### 로컬 Animator 전이 수정
- `Slash1 -> Slash2`: `Exit Time 0.62`
- `Slash1 -> Idle`: `Exit Time 1`
- `Slash2 -> Idle`: `Exit Time 1`
